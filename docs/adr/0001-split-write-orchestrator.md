# 0001 — Split write orchestrator into `serverActionWrite` and `apiWrite`

`executeWrite` in `lib/actions/execute-write.ts` orchestrated five concerns (auth, CSRF, Zod validation, execution, cache invalidation) behind a single 9-field options bag. Two distinct caller profiles — Next.js Server Actions and API route handlers — used disjoint subsets of those fields, making the interface as complex as the implementation. We split it into two focused functions: `serverActionWrite` (positional args: permission, csrfToken, writeFn, options) for server actions and `apiWrite` (positional args: user, writeFn, invalidations) for API handlers. Inlined `action-helpers.ts` (10-line CSRF wrapper, single caller) and moved cache invalidation dispatch into `lib/cache/invalidation.ts`. Removed dead knob `autoRotateCsrf`.
